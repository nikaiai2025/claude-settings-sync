# 設計: Claude Code 設定のGit同期フロー

## 目的
複数端末に散在する Claude Code の設定（Skills/Agents/AGENTS.md/CLAUDE.md）を Git で一元管理し、
ローカル更新の消滅を防ぎながら安全に同期する。

## 管理対象（Windows固定）
ルート: 端末ごとに変わるため自動検出/明示指定する
- `Skills\`（フォルダ）
- `Agents\`（フォルダ）
- `AGENTS.md`（ファイル）
- `CLAUDE.md`（プロジェクト固有設定など）

※ `settings.json` はマシン固有の設定が含まれるため、同期対象外とする。
※ `__pycache__` ディレクトリおよびその中身は、同期対象外とする。

## ルート解決ルール（.claude の場所が端末で違う場合）
優先順で決定する:
1. `--root` 引数（明示指定）
2. 環境変数 `CLAUDE_HOME`
3. 自動検出（`%USERPROFILE%\.claude` など）
4. 既定値 `C:\Users\user\.claude`

## リポジトリ構成（例）
```
repo/
  manifest.yaml
  data/
    Skills/
    Agents/
    AGENTS.md
    CLAUDE.md
  backups/
    2026-02-08_143012_HOSTNAME/
      Skills/
      Agents/
      AGENTS.md
      CLAUDE.md
```

## 同期の基本方針
1. ローカル変更を最優先（消滅を防ぐ）
2. 上書きは常にバックアップ付き
3. `DIFF` は衝突扱いとして、人が正を判断してから反映する

## 主要フロー

### 1. status（確認）
目的: ローカルと Git の差分を可視化
- `Skills/`, `Agents/` 配下のファイル単位でハッシュ比較
- `AGENTS.md` は全文ハッシュ比較
- 結果を次の3種に分類
  - `LOCAL_ONLY`: 同一パスのファイルがローカルにのみ存在
  - `REMOTE_ONLY`: 同一パスのファイルが Git にのみ存在
  - `DIFF`: 同一パスのファイルが両方に存在するが内容差分あり（1行でも違えば DIFF）
- `DIFF` の場合は **内容の近さ** を併記
  - 例: `70/100 lines identical` のように行単位で一致率を表示
  - 目的: どの程度の差分かを即判断できるようにする
- `status` 実行後に詳細差分の表示可否を尋ねる
  - `y` を選ぶと `DIFF` の詳細差分（unified diff）を表示

### 2. collect（ローカル → Git）
目的: ローカル更新を Git 側へ取り込む（上書き含む）
- 前提: `DIFF` は「どちらが正か不明」な状態として扱う
- 方針: **collect は常に強制上書き**
- 実行前の判断
  - `DIFF` の内容を確認し「ローカルが正」と判断した場合のみ実行
- 手順
  1. status を実施（差分・一致率を確認）
  2. `LOCAL_ONLY` は Git 側へコピー
  3. `DIFF` は Git 側へ上書きコピー
  4. `REMOTE_ONLY` は保持（削除しない）
- 付随処理
  - `collect` 後に `git add/commit/push` を自動実行
  - コミットメッセージはタイムスタンプ
- 効果: ローカルが正と判断した場合に確実に Git を更新できる

### 3. apply（Git → ローカル）
目的: Git 側の確定状態をローカルへ反映
- 前提: **実行時に `DIFF` があるのが通常**という運用を採用する
- 方針: **apply は常に強制上書き**
- 実行前の判断
  - `DIFF` の内容を確認し「Git が正」と判断した場合のみ実行
  - 判断を誤った場合に備え **必ずバックアップを作成**
- 付随処理
  - `apply` 前に `git pull` を自動実行
- 実行時のバックアップ
  - 上書き対象ファイルを `backups/<timestamp>_<hostname>/` に保存
- 反映
  - Git 側ファイルをローカルにコピー

## バックアップ仕様
- 対象: **apply 時のみ**（collect は Git 履歴があるため不要）
- 保存先: **プロジェクトフォルダ内** `backups/<timestamp>_<hostname>/...`
- 対象ファイル: 上書きされる全ファイル
- 形式: 元のディレクトリ構造を保持
- 目的: 誤上書きの即時復旧
- 運用: `backups/` は Git から除外（`.gitignore` に追加）

## 更新ルールまとめ
- **ローカル編集後は必ず collect**
- **端末移動直後は apply して最新化してから編集**
- **DIFF は衝突扱い。apply/collect 前に必ず内容を確認して正を判断**
- **apply/collect は常に強制上書き（ただし必ずバックアップ）**

## デグレ防止の要点
- `status` で差分と一致率を必ず確認する運用
- 必要時に詳細差分を出して判断材料を確保する
- 強制時でもバックアップで復旧可能

## 期待される安全な運用例
1. ローカルで更新
2. `status` で差分確認
3. `collect` で Git に反映
4. 他端末で `apply`

---
この設計により「ローカル更新を Git に送る前に apply して消える」事故を防止する。
